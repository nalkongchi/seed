# 🌾 종자검사원 - 종자의 질서를 되찾는 자
> 합격 / 불합격 판정을 통해 왕국의 질서를 되찾는 픽셀 RPG 교육 게임

---

## 📖 게임 소개

**종자검사원 - 종자의 질서를 되찾는 자**은 종자검사 기준을 게임처럼 익힐 수 있도록 만든 픽셀 아트 RPG형 교육 게임입니다.
플레이어는 **종자검사원 지망생**이 되어 6개의 지역을 돌파하고,
각 적이 내세우는 주장에 대해 **합격 / 불합격** 판정을 내리며 최종 보스를 향해 나아갑니다.

현재 버전은 단순한 스테이지 클리어형 구조를 넘어,
- **스토리 모드(모험)**
- **이어하기 / 다시 시작**
- **집중 공부 모드(무한 랜덤 문제풀이)**
- **오답노트 / 엔딩 평가**
를 포함하는 형태로 확장되어 있습니다.

---

## 🎮 주요 특징

### 1) 스토리형 모험 진행
- 타이틀 → 이름 입력 → 오프닝 → 월드맵 → 조우 → 전투 → 클리어 / 게임오버 → 엔딩
- 각 스테이지를 클리어하면 다음 지역이 해금됩니다.
- 마지막 스테이지 **심판의 성**을 돌파하면 엔딩과 최종 평가가 표시됩니다.

### 2) 판정형 전투 시스템
- 플레이어는 적의 주장을 읽고 **합격 / 불합격**을 선택합니다.
- **정답이면 적 HP 감소**, **오답이면 내 HP 감소**
- 플레이어 HP는 항상 **3 고정**입니다.
- 적 HP는 스테이지별로 다릅니다.

### 3) 5문제 제한 제거
- 스토리 전투는 더 이상 “5문제만 풀고 종료”가 아닙니다.
- 각 스테이지 문제 풀을 섞어서 사용하며,
  한 바퀴 다 돌면 다시 섞어서 이어집니다.
- 즉, **누군가의 HP가 0이 될 때까지** 전투가 계속됩니다.

### 4) 조우 일러스트 / 전투 도트 분리
- 조우 화면에서는 큰 적 일러스트가 표시됩니다.
- 전투 화면에서는 별도의 **도트 스프라이트**가 사용됩니다.
- 플레이어도 전투 화면에서 **`player_back.png`** 도트를 사용합니다.

### 5) 집중 공부 모드
- 타이틀 메뉴에서 **집중 공부**를 선택하면,
  스테이지 1~6의 문제를 전부 합쳐 **무한 랜덤 풀이**가 가능합니다.
- 정답 / 오답 / 정답률이 실시간으로 표시됩니다.
- 틀린 문제는 오답노트에 누적됩니다.

### 6) 오답노트 / 설정 / 저장
- 틀린 문제는 자동으로 오답노트에 저장됩니다.
- BGM / 효과음은 게임 내 설정 팝업에서 조절할 수 있습니다.
- 진행 상황은 **localStorage**에 자동 저장되며,
  세이브 데이터가 있으면 타이틀 메뉴가 **이어하기 / 다시 시작** 구조로 바뀝니다.

## ⚔️ 전투 시스템 요약

### 플레이어
- HP: **3 고정**
- 이름: 입력값 사용 (미입력 시 기본값 **용사**)

### 전투 규칙
- 적의 주장 또는 제시 정보를 보고 **합격 / 불합격** 선택
- 정답이면 적 HP 감소
- 오답이면 플레이어 HP 감소
- 결과 팝업 확인 후 다음 문제 진행
- 적 HP가 0이 되면 스테이지 클리어
- 플레이어 HP가 0이 되면 게임오버

### 스테이지 1 튜토리얼
- 1스테이지 첫 전투 전, 전투 규칙 안내 팝업이 1회 표시됩니다.

## 📝 오답노트

- 틀린 문제는 자동 저장됩니다.
- 타이틀의 **오답노트** 버튼으로 언제든 열 수 있습니다.
- 문제 본문, 정답, 해설을 확인할 수 있습니다.
- 초기화 기능도 포함되어 있습니다.

## 💾 저장 시스템

저장 키는 아래와 같습니다.

- 진행 저장: `seedGame_v3`
- 오답노트: `seedGame_wrongnote`
- 사운드 설정: `seedGame_sound`

저장 항목 예시:
- 플레이어 이름
- 스테이지 해금/클리어 상태
- 누적 정답 수 / 오답 수
- 1스테이지 튜토리얼 확인 여부

세이브 데이터가 있을 경우 타이틀 메뉴는 다음처럼 동작합니다.
- **이어하기**: 저장된 진행 상태에서 계속 플레이
- **다시 시작**: 이름 입력부터 새로 시작

## ✏️ 문제 추가 / 수정 방법

문제는 **`data.js`** 에서만 관리합니다.
현재 기본 구조는 아래와 같습니다.

```javascript
{
  text: '"문제 내용을 여기에 씁니다."',
  answer: 'pass',   // 합격이면 pass, 불합격이면 fail
  reason: '판정 근거 설명을 여기에 씁니다.'
}
```

### 규칙
- 각 스테이지의 `questions` 배열에 추가합니다.
- `answer`는 반드시 `'pass'` 또는 `'fail'` 중 하나여야 합니다.
- `reason`은 결과 팝업과 오답노트에 사용됩니다.
- 스토리 모드와 집중 공부 모드 모두 같은 문제 데이터를 공유합니다.

### 추천 팁
- 모험 모드용 대사형 문장과
- 집중 공부용 직접형 문제 문장을
나중에 분리하면 공부 모드 완성도가 더 높아집니다.

1. 이 폴더 전체를 GitHub 저장소에 업로드합니다.
